# 導入手順

## 目的

この手順は、`portable-harness/starter/` を新規プロジェクトへコピーした直後に、何を埋めてから AI 実装を始めるかを示します。

## 最初に埋めるもの

1. `AGENTS.md`
   - `{{PROJECT_NAME}}`
   - `{{SOURCE_OF_TRUTH_PATHS}}`
   - `{{GENERATED_FILES}}`
   - `{{VERIFY_COMMAND}}`
   - `{{OPTIONAL_AUDIT_COMMAND}}`
   - `{{SENSITIVE_DATA_RULES}}`
   - `{{DEPLOYMENT_BOUNDARY}}`
2. `harness/registry.json`
   - verify 対象
   - manual smoke 項目
3. `harness/change_evals.json`
   - 重要機能ごとの eval
4. `harness/phase_gates.json`
   - 完了判定の gate

## 実装開始前の順番

1. `project-os/specs/` に最初の `SPEC` を作る
2. `project-os/tickets/` に対応する `TKT` を作る
3. `implementation_ready` になるまで required_evals と changed_paths を埋める
4. verify コマンドが実行できることを確認する
5. その後に実装を開始する

## 人が見るポイント

- 正本と生成物の境界が埋まっているか
- verify コマンドが具体化されているか
- manual smoke 項目がその案件の重要操作に合っているか
- `project-os/artifacts/TKT-xxxx/` に結果を残す前提ができているか

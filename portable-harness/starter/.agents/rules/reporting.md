# Reporting

## ユーザー向け報告の順番
1. 今回の変更目的
2. これまで不足していた点
3. 今回追加した安全装置
4. 実施した確認
5. まだ人判断が必要な残リスク

## 書き方
- 先に「何の事故を減らす変更か」を書く
- 専門用語は意味を添える
- 実装詳細やファイル名は必要最小限にする
- 失敗時は「どこで止まったか」と「何を守るために止めたか」を平易に書く
- required gate が未達なら、完了表現を使わず open gate 名をそのまま書く

## artifact
- 下書きは `project-os/artifacts/TKT-xxxx/report.md` に残す

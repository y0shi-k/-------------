# PC デザイン言語（Stock Master / Web版）

> **目的**: PC幅（>=1024px）の見た目を、モック画像 **Image #3**（クリーンで余白の広い indigo+白基調）のトーンに統一するための設計正本。
> 今後の PC 各画面（ホーム / 在庫 / 献立・レシピ / 料理・記録 / 設定）の実装・レビューはこの文書を基準にする。
>
> **適用範囲**: PC幅（`@media (min-width: 1024px)`）のみ。スマホ幅（<=640px）の既存レイアウト・操作は温存する（このトーン統一の対象外）。
> **実体**: トークンの正本は `web/src/app/globals.css` の `:root`。本文書はその使い分けと意図を定義する。CSSの実装は各チケットで行う。

---

## 1. デザイン原則

1. **静かな indigo + 白基調**。アクセント色は indigo に一本化し、面で色を塗らない。情報は余白と階層で見せる。
2. **余白を惜しまない**。カードは詰め込まず、縦幅を取ってよい（名前は折り返す、潰さない）。
3. **やわらかい**。角丸とごく弱いシャドウ。濃い枠線・強いグラデーション・原色の多用を避ける。
4. **多色を抑える**。ジャンルタグ等の多色パレットは PC では彩度を落とすか、indigo 系の控えめなチップに寄せる（意味の識別が必要な箇所のみ最小限の色を残す）。
5. **スマホは触らない**。トーン統一は PC専用メディアクエリ内で行い、モバイル既存スタイルへ影響させない。

## 2. デザイントークン

### 2.1 カラー（`:root` 基準 + PC統一用の追加方針）

| 用途 | 変数 / 値 | 備考 |
|------|-----------|------|
| アプリ背景 | `--background` `#f8fafc` | 既存維持。ごく薄いグレー。 |
| サーフェス（カード/バー） | `--surface` `#ffffff` | 既存維持。 |
| 見出しテキスト | `--ink` `#172033` | 既存維持。 |
| 補助テキスト（日付・メタ） | `--muted` `#75849c` | 既存維持。Image #3 のグレー説明文に相当。 |
| 罫線・カード枠 | `--line` `#dbe4f0` | PCでは枠を極力薄く。境界はシャドウ優先。 |
| アクセント（主役） | `--accent` `#4f46e5` | indigo。主要ボタン・active chip・active nav に使う。 |
| アクセント淡（active面） | （追加）`--accent-soft` `#eef0ff` | active nav / active chip の背景。文字は `--accent`。 |
| お気に入り | （追加）`--favorite` `#f43f5e` | ハート（rose）。塗りは選択時のみ、未選択はアウトライン。 |
| 完了・成功 | `--green` `#009f73` | 既存。完了バッジ等。多用しない。 |
| 注意・警告 | `--warning` `#f59e0b` | 既存。期限・今日など最小限。 |
| ソフトシャドウ | （追加）`--shadow-card` `0 8px 24px rgb(15 23 42 / 6%)` | カードの浮き。濃くしない。 |

> 追加トークン（`--accent-soft` / `--favorite` / `--shadow-card`）は、最初の土台チケットで `:root` に追加する。

#### PCで抑制する既存表現
- 献立スケジュールの **sky-blue 多用**（`#0ea5e9` / `#bae6fd` 系）→ indigo 基調 + ニュートラルへ寄せる。
- ジャンルタグの **7色パレット**（`recipe-genre-pill[data-palette]`）→ PCでは彩度を落とした淡色 or indigo 系チップに統一。意味の識別が要る場合のみ最小限。
- 濃いめの枠線・二重シャドウ → `--line` を薄く、`--shadow-card` 一本に。

### 2.2 タイポグラフィ

| 役割 | サイズ | ウェイト | 色 |
|------|--------|----------|-----|
| 画面見出し（ようこそ等） | 1.5–1.75rem | 800 | `--ink` |
| セクション見出し（最近作ったレシピ等） | 1.05–1.15rem | 800 | `--ink` |
| カードタイトル（レシピ名） | 0.95–1rem | 700 | `--ink`（2行まで折り返し可、`-webkit-line-clamp: 2`） |
| 本文・メタ・日付 | 0.72–0.8rem | 600–700 | `--muted` |
| eyebrow / ラベル | 0.66rem | 900 | `--muted`（letter-spacing 0.12em） |

- カードタイトルは **1文字省略（「究…」）を禁止**。最低でも2行表示できる高さを確保する。

### 2.3 角丸・シャドウ・スペーシング

| 項目 | 値 |
|------|-----|
| カード角丸 | 16px |
| ボタン角丸 | 12px |
| チップ/検索/アバター角丸 | 999px（pill / 円） |
| カード影 | `--shadow-card` |
| カードグリッド gap | 16–20px |
| カード内padding | 14–16px |
| カラム数（レシピ等） | PC幅で 4 列目安（`repeat(auto-fill, minmax(220px, 1fr))`） |

## 3. コンポーネント規定（Image #3 準拠）

### 3.1 サイドバー
- 白地。`アイコン + ラベル` の縦リスト。
- active 項目: 背景 `--accent-soft`、アイコン/文字 `--accent`。非activeは `--muted`。
- 上部にブランド（ロゴ + サブコピー）。下部に設定（ギア）。

### 3.2 トップバー
- 検索は pill 入力（薄グレー背景）。右に通知・アカウント（円アバター）。
- ステータス/同期表示は控えめなチップで。色面で主張しない。

### 3.3 フィルタチップ（カテゴリ）
- pill 形。active = `--accent` 塗り + 白文字。非active = 白地 + `--line` 枠 + `--muted` 文字。
- 例: すべて / お気に入り / よく作る / 和食 / 洋食 / 中華 …（データ由来で生成）。
- 右端に並び替えドロップダウン。

### 3.4 セクションヘッダ
- 左: セクション見出し（800）。右: `もっと見る ›` のテキストリンク（`--accent`）。

### 3.5 レシピカード（今回の主対象）
- **縦型カード**。上から順に:
  1. **ビジュアル領域**（カード上部、横長 4:3 目安、角丸上）。当面は **写真なしプレースホルダ**（淡い背景 + 料理アイコン/頭文字）。将来の写真対応の差し込み口にする。
  2. **レシピ名**（700、最大2行、省略は2行クランプ）。
  3. **メタ行**（`--muted`、小）: 当面は `材料◯品目・調理◯回・登録日` 等の既存情報。**調理時間は今回出さない**。
  4. **フッタ行**: 左=補助メタ、右=**お気に入りハート**（選択時 `--favorite` 塗り、未選択アウトライン）。
- **操作ボタン（料理する/編集/削除）** はタイトル行に詰め込まない。ホバー時表示 or カード下部/ケバブメニューに退避し、名前の表示幅を確保する。
- グリッドは 4 列目安、gap 16–20px。

### 3.6 プライマリボタン
- indigo 塗り（`--accent`）、白文字、角丸 12px、アイコン + ラベル。
- 「+ 新規レシピ / 新しいレシピを追加」等の主要アクションに使用。

| 項目 | 決定 | 影響 |

## 5. 段階適用の順序（チケット計画の骨子）

1. **土台**: `:root` に `--accent-soft` / `--favorite` / `--shadow-card` を追加。PC共通の角丸・影・カードの基準クラスを整える。
2. **レシピカード**（今回の主訴）: §3.5 の縦型カードへ。名前折り返し・操作ボタン退避・プレースホルダ領域・お気に入りハート枠（機能は §4 のお気に入り新設チケットと連動）。
3. **お気に入り schema**（危険変更・別チケット）: `recipes.is_favorite` 追加 + RLS 確認 + 保存処理 + カテゴリチップ「お気に入り」連動。
4. **献立スケジュール**: sky-blue 抑制、indigo 基調へ（TKT-0165 の縦アジェンダを踏襲しつつ配色をトーンに合わせる）。
5. **在庫 / 料理・記録 / 設定 / ホーム**: 同トークンで順次トーン統一。

> 各ステップは独立チケット。危険変更（お気に入り schema）以外は非危険（CSS/JSX 軽微）に保つ。

---

## 6. スマホとの関係
- 本トーン統一は PC専用メディアクエリ内に閉じる。モバイルの既存レイアウト・操作・配色は変更しない。
- 共有する `:root` トークン追加は両者に影響しうるため、追加時はモバイル表示の回帰を verify で確認する。

## 7. 未決事項
- ジャンルタグの「彩度を落とす」具体値（淡色 or indigo 単色化）は、レシピカード実装時に実画面で確定する。
- 将来のレシピ写真（履歴写真流用 or 表紙写真新設）は本文書の差し込み口を前提に別途検討。

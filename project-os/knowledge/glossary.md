# Glossary（用語集・ドメイン知識）

> Stock Master（料理レシピ・食材管理アプリ）のドメイン用語と判断基準。表記・解釈のブレを防ぐ正本。
> `/glossary` スキルがこのファイルを参照して答える。

## 用語

| 用語 | 意味 | 対応データ（Supabase） |
|---|---|---|
| 在庫 | ユーザーが現在持っている食材・調味料 | `inventory_items` |
| 登録待ち / staging | これから在庫に入れる登録候補。AI解析→一括登録の前段 | `staging_items` |
| 買い物リスト / 買い足し | 不足している・購入予定の食材 | `shopping_items` |
| レシピ | 手動登録またはAI抽出した料理の作り方 | `recipes` |
| レシピ材料 | レシピを構成する食材（検索可能） | `recipe_ingredients` |
| レシピ候補 | 在庫から作れる料理案（買い足しが少ない案を優先） | （導出） |
| 献立 / 献立スケジュール | 朝・昼・晩スロットに割り当てた予定 | `meal_schedules` |
| 調理履歴 | 作った料理の記録（写真・評価・感想） | `cooking_history` |
| 消費期限 / 実質期限 | 食材を使い切る目安（AIが推測する場合がある） | `inventory_items` の期限系カラム |
| 写真 | ユーザーの非公開画像（食材スキャン・完成写真） | `photos`（Storage 非公開） |

## 判断基準（プロダクトの美学）

- 食材登録は**手入力より写真認識を優先**する（継続利用の負担を下げる）。
- 料理提案では、**買い足しが少ない案を優先**する。
- ユーザーに難しい設定・細かいカテゴリ選択を要求しない。
- AI解析の結果は**確定せず「登録候補(staging)」に置く**（ユーザーが確認・修正できる）。
- 写真は個人情報を含みうるため、**非公開バケット・推測不能URL**を標準にする。

## 目的

モードBに「AIレシピ考案」機能を追加する。在庫の期限が迫っている食材を優先的に使うレシピと、ユーザーが指定した食材から作れるレシピをGemini APIで生成し、レシピ集へ保存できるようにする。

## タスク

- [ ] モードB内に「AI考案」サブタブ/セクションUIを追加
- [ ] 「🚨 優先消費レシピを考案」ボタンを実装
  - [ ] 期限3日以内の在庫食材を自動抽出するロジック
  - [ ] Geminiプロンプト作成（期限食材リスト＋JSON出力指定）
  - [ ] API送信＆レスポンスパース
- [ ] 「🎯 指定食材から考案」ボタンを実装
  - [ ] 在庫一覧から食材を選択するUI（チェックボックス）
  - [ ] Geminiプロンプト作成（指定食材リスト＋JSON出力指定）
  - [ ] API送信＆レスポンスパース
- [ ] 生成レシピのプレビューモーダルを実装（レシピ名、材料リスト、手順リストの表示）
- [ ] 「レシピ集に保存」ボタンでTKT-0004の保存フローへ連携
- [ ] APIエラー時のエラーハンドリング（showToast、フリーズ防止）
- [ ] verify コマンド実行
- [ ] artifacts を `project-os/artifacts/TKT-0005/` に作成

## Acceptance

- モードBの「AI考案」で「優先消費レシピ」を押すと、期限3日以内の食材がプロンプトに含まれてレシピが生成される
- 在庫から食材を選んで「指定食材から考案」を押すと、その食材を含むレシピが生成される
- 生成レシピをプレビュー後、「レシピ集に保存」でレシピ集シートに追加される
- APIエラー時にエラートーストが表示され、操作可能な状態に戻る
- 期限迫る食材が0件の場合は適切なメッセージが表示される
- HTML構文チェックが通る
- `executeGAS` と `GAS_URL` が残っている

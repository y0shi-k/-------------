# Summary

全画面ビュー（TKT-0201）で実装したサブグルーピングUIを、レシピ編集モーダルにも展開する。選択/グルーピング/解除/ラベル導出の共通ロジックを流用し、編集画面の材料・調味料行で同じ操作ができるようにする。保存は既存 `saveRecipe` フローで group_index を永続化する。

## 背景

要望は「編集・全画面ビューどちらも」サブグルーピングできること。TKT-0201 が全画面ビュー、本チケットが編集画面を担う。

## 仕様

- 対象は `web/src/components/recipe-meal-workspace.tsx`（編集モーダル ~L2336〜2438）と `globals.css`。
- TKT-0201 で作った選択state・グルーピング/解除・ラベル導出関数を共通化して再利用する（重複実装を避ける）。
- 編集モーダルの材料・調味料行に選択UI（クリック/Cmd・Ctrl）を付ける。行内入力やD&Dハンドルとイベントが競合しないようにする。
- 「グルーピング」/「グループ解除」ボタンを各見出し（材料/調味料）隣に条件表示する。
- 保存は既存 `saveRecipe`（TKT-0200で group_index 対応済み）に乗せる。新規DB書き込み機構は追加しない。
- TKT-0198 の編集画面D&Dとグループ境界（グループ内/グループ間移動の group_index 更新）を整合させる。

## 非対象

- 全画面ビュー側のグルーピングUI（TKT-0201で実装済み）
- 任意グループ名（自動ラベルのみ）
- DB schema変更

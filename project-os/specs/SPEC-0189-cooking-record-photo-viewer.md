# Summary

料理記録の複数写真を閲覧する導線を追加する。カレンダー/タイムラインのカードに枚数バッジを出し、写真タップで
全画像・評価・コメントを読み取り専用で見られる閲覧モーダルを新設する。既存の signed_url を表示するだけの非危険変更。
閲覧モーダルから既存の編集モーダルへ遷移できる。

## 背景

編集モーダルで複数写真を登録できるようになったが、(1) カレンダー/タイムラインのカードは写真1枚しか表示されず複数あることに
気づけない、(2) 全画像を見るには編集を開くしかなく閲覧専用の場所がない、という問題がある。

## 仕様

- プロジェクト名: Stock Master（料理レシピ・食材管理アプリ）
- 現役正本（編集対象）: `web/`
- 対象コンポーネント: `web/src/components/cooking-history-board.tsx`
  - `HistoryPhoto`: signed_url を持つ写真が2件以上のとき枚数バッジを重ね、写真をクリック可能化（onView）。
  - `CookingHistoryBoard`: `viewingItem` state ＋ 読み取り専用の閲覧モーダル（全画像グリッド＋`renderStars(rating)`＋note、フッタに「編集」＝既存 onEdit 流用と「閉じる」）。
  - カレンダーセル/タイムライン両方のカードで写真クリック→onView を配線。
- 対象CSS: `web/src/app/globals.css`（枚数バッジ、閲覧モーダル、スマホ幅対応）
- モーダルは既存 `modal-backdrop` / `canvas-modal` / `modal-close-button` パターンに合わせ、`role="dialog" aria-modal="true"`、背景クリックで閉じる。
- 署名URLは `item.photos[].signed_url`（ボード読み込み時に付与済み）。新規 fetch 不要。
- verify: `/verify`

## 非対象

- 編集モーダルの写真UI改善（既存×削除・新規サムネ）（→ SPEC-0188）。
- 画像の拡大ズーム/スワイプギャラリー等の高度な閲覧（まずはグリッド表示で成立）。
- 写真Storage / schema / RLS / バケット設定の変更。

## Acceptance Example

- 複数写真の料理記録で、カードに枚数バッジが出て写真タップで閲覧モーダルが開き、全画像・評価・コメントが見えることを確認し report に記録する。
- 閲覧モーダルから「編集」で編集モーダルへ遷移できることを確認する。

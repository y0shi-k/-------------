## Summary

モードBに「AIレシピ考案」機能を追加する。在庫の実質期限（または表示期限）が迫っている食材を自動抽出してプロンプトに含めた「優先消費レシピ」と、ユーザーが在庫から選んだ食材を必須材料として指定した「指定食材レシピ」の2つのモードを提供する。

## 背景

ユーザーは「何を作ろう」という意思決定を毎日行う必要がある。在庫の期限が迫っている食材を優先的に使うレシピをAIが提案することで、食品ロスを減らし、買い物の負担も軽減する。また、既に持っている特定の食材（例: 豚肉と白菜）から作れるレシピを知りたい場面も多い。

## 仕様

### UI構成

- モードB内に「AI考案」サブタブ/セクションを設置
- 2つのボタン: 「🚨 優先消費レシピを考案」「🎯 指定食材から考案」

### 優先消費レシピ

1. フロントエンドで `state.inventory` を走査し、実質期限（未設定なら表示期限）が3日以内の食材を自動抽出
2. 抽出した食材リストをGeminiプロンプトに含める
3. プロンプト例:
   ```
   以下の食材を優先的に使うレシピを1つ提案してください。
   期限が近い食材: [ {name, qty, unit}, ... ]
   
   他に必要な材料も含めて、以下のJSONで出力:
   {
     "name": "レシピ名",
     "ingredients": [{"name":"", "amount":0, "unit":""}, ...],
     "steps": ["手順1", "手順2", ...]
   }
   ```
4. 生成結果をプレビューモーダルで表示
5. 「レシピ集に保存」ボタンでTKT-0004の保存フローへ渡す

### 指定食材レシピ

1. 在庫一覧からユーザーが複数食材をチェックボックスで選択
2. 「レシピを考案」ボタンでGeminiへ送信
3. プロンプト例:
   ```
   以下の食材を必須材料として含むレシピを1つ提案してください。
   指定食材: [ {name, qty, unit}, ... ]
   
   出力形式は同上
   ```
4. プレビュー→保存フロー

### プレビュー・保存フロー

- 生成結果はモーダルで表示。材料リストと手順を一覧表示
- 「レシピ集に保存」ボタンで、TKT-0004のレシピ保存関数（または同等のGASペイロード）を呼び出す
- 保存成功後、showToastで通知し、レシピ集一覧へ誘導

### エラーハンドリング

- APIエラー時はshowToastでエラー表示、フリーズしない
- 期限迫る食材が0件の場合は「期限が近い食材がありません」と表示

## 非対象

- 献立スケジュールへの直接割り当て（TKT-0006で対応）
- 買い物リストへの不足材料抽出（TKT-0007で対応）
- レシピ集CRUD自体（TKT-0004で対応）

## Acceptance Example

- モードBの「AI考案」で「優先消費レシピ」を押すと、期限3日以内の食材がプロンプトに含まれてレシピが生成される
- 在庫から食材を選んで「指定食材から考案」を押すと、その食材を含むレシピが生成される
- 生成レシピに「レシピ集に保存」を押すと、レシピ集シートに追加され一覧に表示される
- APIエラー時にエラートーストが表示され、操作可能な状態に戻る

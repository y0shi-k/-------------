# Summary

期限が近い食材を「一覧で警告する」だけでなく、「何に使うか」までつなげる使い切り導線を追加する。食品ロス削減をこのアプリの中核価値として前面に出す。

## 背景

在庫管理アプリで重要なのは、期限を知ることだけではなく、期限食材を使って次の料理に変えること。競合アプリでも在庫/期限/レシピ提案の結合が差別化ポイントになっている。Stock Master は在庫、レシピ、AI提案、献立を既に持っているため、使い切り導線を強化しやすい。

## 仕様

- 期限食材リストを派生する。
  - 基準日: 今日
  - 期限日: `item.limit2 || item.limit`
  - グループ: 期限切れ、3日以内、7日以内、期限未設定
  - 並び: 期限切れ、近い順、保存場所、品名
- 食材カードに表示する情報:
  - 品名、数量/単位、分類、保存場所
  - 期限日と残日数
  - 実質期限か表示期限か
  - 開封日がある場合は開封済み表示
- 導線:
  - 「この食材で探す」: レシピ検索を食材検索モードに切り替えて該当食材で検索する。
  - 「AIで使い切り」: 既存の優先消費AI提案に対象食材を渡す。
  - 「買い物リスト確認」ではなく、まず消費/献立へつなげる。
- 今日ダッシュボードには上位数件だけ表示し、詳細は食材管理内の専用表示に移動する。

## 非対象

- 期限通知
- 栄養/カロリー
- 期限推定AIの全面再設計
- 新規Spreadsheet列追加

# Summary

レシピ単位と食材在庫の単位が違う場合でも、料理完了時に実際の在庫単位へ揃えて減算できるようにする。

## 仕様

- 食材在庫シートに `単位換算JSON` 列を追加する。
- `unitConversion` は `{"fromQty":1,"fromUnit":"パック","toQty":150,"toUnit":"g"}` の形で扱う。
- 換算は双方向に使う。例: `1パック=150g` は `150g→1パック` と `1パック→150g` の両方に対応する。
- 換算未設定で単位が違う場合は、今回だけの在庫単位入力欄を出し、その入力値だけを減算する。
- 食材編集/登録画面に `1 [単位A] = [数量B] [単位B]` の入力欄を追加する。

## 非対象

- AIによる自動推測換算
- 換算未設定時の自動減算
- 操作ごとの即時GAS同期

## Acceptance Example

- 在庫 `豆腐 3パック`、換算 `1パック=150g`、レシピ `豆腐 150g` なら `2パック` になる。
- 在庫 `豆腐 450g`、換算 `1パック=150g`、レシピ `豆腐 1パック` なら `300g` になる。
- 換算未設定で在庫 `豆腐 3パック`、レシピ `豆腐 150g` の場合、今回消費量 `1パック` を入力して `2パック` にできる。

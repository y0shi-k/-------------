# SPEC-0019-cooking-viewer-ui

## 背景

Phase3 モードC（クッキングビューア）の基盤となる大画面UIを構築する。レシピの材料と手順を料理中に遠くからでも読める大きな文字と色分けバッジで表示する。

## 変更範囲

- `app.html` のモードCセクション（`#modeC`）を新規実装
- 献立スケジュールのスロットに「開始」導線を追加
- レシピ集カードに「料理する」導線を追加

## 詳細仕様

### 1. レシピ選択導線

- **献立スケジュール**: 割り当て済みスロットに「▶ 開始」ボタン（小さく）を配置。タップで `openCookingViewer(recipeId)` を呼ぶ。
- **レシピ集**: レシピカードに「料理する」ボタンを追加（またはカードの三点リーダーメニュー内）。
- **直接**: モードCに切り替えた時、現在の献立（今日の予定）があれば自動で表示。なければ「献立を選んでください」ガイド表示。

### 2. クッキングビューアレイアウト

- ヘッダー: レシピ名（`text-2xl font-bold`）
- タブ切替: 「材料」「手順」の2タブ（または1画面に上下で並べる）
- 材料タブ:
  - 各材料を1行ずつ大画面カードで表示
  - 品名（`text-xl`）と分量（`text-lg font-semibold`）
  - 分類バッジ: 「食材」→ 緑系（`bg-green-100 text-green-800`）、「調味料」→ 黄系（`bg-yellow-100 text-yellow-800`）
  - 分量バッジ: 「大さじ」→ 赤系（`bg-red-100 text-red-800 font-bold`）、「小さじ」→ 青系（`bg-blue-100 text-blue-800 font-bold`）
- 手順タブ:
  - ステップ番号付きの大画面カード
  - 手順テキスト内の分量表現（正規表現で「醤油 大さじ1」などを検出）を強調表示（`text-red-600 font-bold` など）
  - インライン分量は `bg-gray-100 px-1 rounded` で視覚的に区切る

### 3. 完了ボタン

- 画面下部に固定配置（`fixed bottom-20 left-0 right-0` など、ボトムナビより上）
- ラベル: 「料理を完了する」
- タップで `openConsumptionAdjustmentModal(recipeId)` を呼ぶ

## 技術的制約

- `alert` / `confirm` / `prompt` を使わない
- 新規モーダルは既存のモーダルパターン（`hidden` クラス切替）に従う
- 材料JSONと手順JSONのパース失敗時は `showToast('レシピデータの読み込みに失敗しました')` で保護

## テスト観点

- 献立スケジュールの「開始」ボタンでビューアが開く
- 材料が大きく読める
- 大さじ・小さじが色分けされている
- 手順内の分量が強調されている
- verify が通る

## Summary

モードBの基盤となる「レシピ集」機能を実装する。レシピの閲覧・追加・編集・削除と、WebやメモからコピーしたテキストをGemini APIで構造化してレシピ集へ登録する機能を含む。

## 背景

Phase2では献立プランナーを構築するが、その前提として「レシピ集」データを管理するUIが必要。ユーザーは既存のレシピを貯め、そこから献立に割り当てる。手動入力だけでなく、Webレシピやメモのコピペからの自動構造化も必須。

## 仕様

### データベース

- 対象シート: `レシピ集`（シート②）
- ヘッダー: `レシピID`, `レシピ名`, `材料JSON`, `手順JSON`, `出典`, `調理回数`, `調理日履歴`
- 初期化時（handleInit）にヘッダー行を追加していない場合は追加する

### UI構成

- モードB内に「レシピ集」サブタブ/セクションを設置
- レシピ一覧: カード型または行リスト型でレシピ名、主要材料数、調理回数を表示
- 各レシピカードに「編集」「削除」ボタン
- フローティングまたはヘッダーに「＋新規レシピ」「📋テキストから追加」ボタン

### レシピ追加フロー（手動）

1. モーダル表示（タイトル入力、出典入力）
2. 材料を行ごとに入力（品名、数量、単位）。＋ボタンで行追加
3. 手順を行ごとに入力。＋ボタンで行追加
4. 保存時に `材料JSON` / `手順JSON` をJSON文字列化してGAS経由でレシピ集シートへappend

### レシピ追加フロー（テキスト貼り付け＋Gemini構造化）

1. テキストエリアにレシピテキストを貼り付け
2. 「AIで構造化」ボタンでGemini APIへ送信
3. プロンプト: `以下のレシピテキストから、レシピ名・材料一覧（name/amount/unit）・手順リストを抽出し、以下のJSONで出力...`
4. レスポンスをパースし、モーダル内フォームに反映
5. ユーザーが確認・修正後、保存

### レシピ編集フロー

- 既存レシピの材料JSON・手順JSONをパースして編集モーダルに展開
- 変更後、該当行をシート上で更新（IDで行特定）

### レシピ削除フロー

- 確認ダイアログ（showToastまたはDOMベース）後、該当行を削除

### GASペイロード

- `action=init` 時に `レシピ集` シートのヘッダー確認・追加
- `action=saveRecipe`: 新規追加（appendRow）
- `action=updateRecipe`: 更新（IDで行検索してsetValues）
- `action=deleteRecipe`: 削除（IDで行検索してdeleteRow）
- `action=getRecipes`: 全件取得（一覧表示用）

## 非対象

- 献立スケジュールへの割り当て（TKT-0006）
- 買い物リスト連携（TKT-0007）
- AIによる自動レシピ考案（TKT-0005）

## Acceptance Example

- モードBに切り替えて「レシピ集」タブを開くと、既存レシピが一覧表示される
- 「＋新規レシピ」からレシピを手動追加し、一覧に反映される
- 「📋テキストから追加」にレシピブログのコピペを貼り、AI構造化後に保存できる
- 編集モーダルで材料・手順を変更し、反映される
- 削除後、一覧から消える

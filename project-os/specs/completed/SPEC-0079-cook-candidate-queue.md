# Summary

「今すぐ日付は決めないが、近いうちに作りたい料理」を一時的に置く候補キューを追加する。献立に直接入れる前の中間置き場を作り、毎日の献立決めの負担を減らす。

## 背景

献立スケジュールは日付と食事枠を決める操作なので、気軽な保存先としては少し重い。レシピ管理アプリでは「今週作りたい」「あとで作る」「献立キュー」のような中間状態があると、買い物前・週末計画・期限食材消費の整理がしやすい。

## 仕様

- 候補キューはレシピ単位で扱う。
- 候補に持たせたい表示情報:
  - レシピID、レシピ名
  - 追加理由: 手動、期限食材、AI提案、履歴由来
  - 期限食材の根拠がある場合は食材名と期限
  - 追加日時または並び順
- 永続化案:
  - 第一候補: 既存の献立スケジュールに `食事区分=候補` または `予定日` に安全な未割当表現を使う案。ただし既存スケジュール表示やGAS読み書きとの相性を実装前に確認する。
  - 第二候補: レシピの既存JSON系メタに保存できるか確認する。ただしスキーマ外の列追加はしない。
  - どちらも不自然なら、ローカル表示だけのMVPにして、永続化は別ticketで再検討する。
- UI:
  - レシピカードに「作りたい」トグル/ボタンを追加する。
  - レシピ集または今日ダッシュボードに候補キュー一覧を表示する。
  - 候補カードから「献立に入れる」「料理する」「外す」ができる。
  - 期限食材由来の候補は根拠を表示する。

## 非対象

- 家族共有
- 通知
- 新規Spreadsheetシート追加
- AIによる週間献立の完全自動確定

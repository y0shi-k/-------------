# Summary

レシピ集画面に検索フィルタを追加し、ユーザーが目的のレシピを素早く見つけられるようにする。

## 背景

現在のレシピ集は `state.recipes` を全件表示するのみで、レシピ数が増えると目的のレシピを探すのが困難になる。

## 仕様

### UI構成

`#bTabRecipes` 内のボタン行（新規レシピ・テキスト追加）の下に、以下を追加する：

1. **セグメントボタン**: 「レシピ名」「食材」「すべて」の3モード切り替え
2. **テキスト入力**: 検索キーワード入力（リアルタイムフィルタ）

### 検索ロジック

`renderRecipeList()` を修正し、以下のフィルタを追加する：

- **レシピ名モード**: `recipe.name` にキーワードが含まれるレシピのみ表示
- **食材モード**: `recipe.ingredients`（JSON配列）内の各食材 `name` にキーワードが含まれるレシピのみ表示
- **すべてモード**: レシピ名 **または** 食材名にキーワードが含まれるレシピを表示

検索は case-insensitive（小文字変換後に includes）。

### 空状態

- 検索結果が0件かつ `state.recipes.length > 0` の場合：「該当するレシピが見つかりません。」
- `state.recipes.length === 0` の場合：既存メッセージ「レシピが登録されていません...」を維持

### スタイル

- セグメントボタン: `shopping-view-btn` と同じパターン（`bg-slate-100 rounded-xl p-1 flex gap-1`）
- 入力欄: `scheduleRecipeSearch` と同じパターン（`w-full bg-slate-50 border-2 border-slate-100 rounded-2xl px-4 py-3 text-sm font-bold`）
- 検索エリア全体: `bg-white rounded-2xl p-4 shadow-sm border border-slate-100 space-y-3`

## 非対象

- サーバーサイド検索（クライアント側フィルタのみ）
- 検索履歴・お気に入り検索の保存
- タグやカテゴリによる絞り込み
- 並び替え機能（別途検討）

## Acceptance Example

1. ユーザーがレシピ集を開く
2. 検索エリア（セグメントボタン＋入力欄）が表示されている
3. 「レシピ名」を選択し、「カレー」と入力 → カレー関連レシピのみ表示
4. 「食材」を選択し、「玉ねぎ」と入力 → 玉ねぎを含むレシピのみ表示
5. 「すべて」を選択し、「チキン」と入力 → レシピ名または食材に「チキン」を含むレシピが表示
6. 入力をクリア → 全件表示に戻る
7. 存在しないキーワードを入力 → 「該当するレシピが見つかりません。」と表示

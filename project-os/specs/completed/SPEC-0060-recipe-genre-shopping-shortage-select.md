# Summary

レシピ集にジャンル列を追加し、候補チップ付きの複数タグUIを提供する。献立スケジュールへレシピを追加した時の不足材料自動追加は、候補選択モーダルでユーザーが確認してから買い物リストへ積む。

## 仕様

- レシピ集シートの後続列に `ジャンル` を追加する。
- `ジャンル` 列には `["和食","作り置き"]` のようなJSON配列文字列を保存する。
- 既存レシピの単一ジャンル文字列は、読み込み時に1要素タグ配列として扱う。
- 初期候補は `和食`, `洋食`, `中華`, `韓国`, `イタリアン`, `エスニック`, `その他`。
- ジャンル候補は初期候補と既存レシピの `genre` タグ配列から重複なしで組み立てる。
- 候補外のジャンルは、レシピ編集欄の入力からそのレシピへ直接タグ追加する。
- スケジュール追加時の不足候補は既存の在庫比較ロジックを使い、数量不足も在庫ゼロも対象にする。
- 買い物候補モーダルは `ALL` / `食材` / `調味料` タブとチェックボックスを持ち、初期状態は全件オフ。

## 非対象

- ジャンル候補専用シート・永続設定
- 買い物候補の数量編集
- レシピ詳細からの買い物追加フロー変更

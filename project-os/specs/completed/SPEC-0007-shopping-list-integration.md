## Summary

献立にレシピを割り当てた際、またはレシピ集から直接、レシピに必要な材料と現在の食材在庫を比較して不足分を買い物リストに自動追記する機能を実装する。追記後、ユーザーはワンタップで買い物リストを確認できる。

## 背景

献立を決めた後に「このレシピを作るには何が足りない？」を手動で確認するのは手間。レシピの材料JSONと在庫を自動比較し、不足分を買い物リストに追記することで、買い物の計画を効率化する。

## 仕様

### トリガー箇所

1. **献立割り当て時**（TKT-0006の `assignMeal` 成功後）
2. **レシピ集からの直接抽出**（レシピ詳細モーダルに「買い物リストに追加」ボタンを設置）

### 比較ロジック

1. レシピの `材料JSON` をパース: `[{name, amount, unit}, ...]`
2. 現在の `state.inventory` を走査し、同じ `品名`（完全一致または部分一致）で在庫があるか確認
3. 比較ロジック:
   - 在庫に同名アイテムがない → 不足としてリストに追加
   - 在庫に同名アイテムがあるが数量が足りない → 差分をリストに追加
   - 在庫に同名アイテムがあり数量が十分 → 追加しない
   - 単位が異なる場合（g vs 個など）→ 現時点では別アイテムとして扱う（単位換算はPhase2以降で検討）
4. 不足リストを構築: `[{name, amount, unit, recipeName}]`

### 買い物リスト追記

1. 不足リストをGASへ送信: `{action:'addToShopping', items:[...]}`
2. GAS側で `買い物リスト` シートに追記
   - 同名・同単位の未購入アイテムがあれば `必要数量` を加算
   - なければ新規行追加（リストIDはUUID生成）
   - ステータスは「未購入」
   - `紐づくレシピ名` にレシピ名を設定
3. 追記後、GASから更新された買い物リスト全件を返す
4. フロントエンドの `state.shopping` を更新

### ジャンプ導線

- 追記完了後、showToast で「不足材料が買い物リストに追加されました」と通知
- トースト内またはトースト下に「買い物リストを確認」ボタンを表示
- ボタンタップで `switchMode('A')` 後、`filterByLocation('買い物リスト')` を実行して自動遷移

### 手動追加ボタン（レシピ詳細から）

- レシピ集のレシピ詳細モーダルに「🛒 買い物リストに追加」ボタンを設置
- タップで上記の比較→追記フローを実行

### GASペイロード

- `action=addToShopping`: 不足材料の一括追記（加算 or 新規）
- `action=getShopping`: 買い物リスト全件取得（更新後の再読み込み用）

## 非対象

- 単位換算（g↔個など）は行わない（別フェーズで検討）
- 食材の類似名マッチング（「人参」と「にんじん」など）は完全一致のみ（別フェーズで検討）
- 献立スケジュールUI自体（TKT-0006）
- レシピ集CRUD（TKT-0004）

## Acceptance Example

- 献立に「カレー」（材料: 豚肉300g, 玉ねぎ2個, 人参1本）を割り当てる
- 在庫に玉ねぎ1個しかない場合、買い物リストに「玉ねぎ 1個」「豚肉 300g」「人参 1本」が追加される（または玉ねぎは差分1個）
- 追加完了後、「買い物リストを確認」ボタンが表示され、タップでモードAの買い物リストタブに遷移する
- 買い物リストに既に「豚肉 200g（未購入）」がある場合、「豚肉 300g」が追加された結果、合計500gになる（または新規行として追加されて一覧に表示される）

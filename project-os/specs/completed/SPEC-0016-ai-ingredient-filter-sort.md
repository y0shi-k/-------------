# SPEC-0016: AI食材選択モーダル拡張（検索・ソート・フィルタ・ジャンル分類）

## 概要
AI献立・レシピ機能の「指定食材から」で開く食材選択モーダルに、検索・ソート・フィルタ機能を追加する。また、在庫管理の分類（ジャンル）選択肢を拡張し、食材選択モーダルでもジャンルフィルタを使えるようにする。

## 変更範囲
- `app.html` のみ（単一HTMLアプリ）

## 詳細仕様

### 1. 在庫管理側：分類選択肢の拡張
- 対象：`<select id="f-type">`
- 既存の「食材」「調味料」はそのまま残す
- 追加オプション：
  - `野菜`
  - `肉`
  - `魚`
  - `乳製品`
  - `穀物`
  - `その他`

### 2. 食材選択モーダル（`aiIngredientSelectModal`）拡張

#### UI追加要素（モーダル内リストの上部）
1. **検索欄**
   - テキスト入力。placeholder: "食材名で検索..."
   - `oninput` でリアルタイム絞り込み
2. **ソート切り替えボタン群**
   - 対象キー：`name`（名前）, `buy`（購入日）, `limit2`（期限）
   - 既存の `setSort` / `updateSortButtons` パターンと同様のUI（タブ風ボタン + 昇降矢印）
   - モーダル専用の一時状態で管理（`state` 本体の `sortBy`/`sortOrder` は在庫管理側と干渉しない）
3. **保存場所フィルタ**
   - `<select>` で既存の保存場所（+「すべて」）から選択
4. **ジャンル（分類）フィルタ**
   - `<select>` で分類（+「すべて」）から選択

#### 表示ロジック
- `renderAiIngredientList()` を拡張
- まず検索テキストで `item.name` を部分一致絞り込み
- 次に保存場所フィルタ（`loc`）で絞り込み（「すべて」以外の場合）
- 次にジャンルフィルタ（`type`）で絞り込み（「すべて」以外の場合）
- 最後にソートを適用
- 既存の期限に応じた色分け（期限切れ=赤、3日以内=黄）もそのまま保持

### 3. データ構造
- 既存の `state.inventory` 各要素の `type` フィールドを使用
- 新規フィールド追加なし（既存スキーマ互換）

## 非機能要件
- 既存の在庫管理側の `sortBy`/`sortOrder` と干渉しないこと
- モーダルを閉じた際に、フィルタ状態はリセットされても構わない（シンプルに）
- Canvas環境追加チェックに抵触しないこと（alert/confirm/promptなし、showToast使用）

## テスト観点（verify）
1. `python3 -c "import html.parser..."` が通る
2. `showToast` 関数が存在する
3. `alert`/`confirm`/`prompt` が残存していない
4. 新規スプシ書き込みコードが `syncPendingChanges()` 以外で個別 `executeGAS` していない
5. `f-type` の `<option>` が拡張されている
6. `aiIngredientSelectModal` 内に検索・ソート・フィルタ要素が存在する
7. `renderAiIngredientList` が検索・フィルタ・ソートを考慮している

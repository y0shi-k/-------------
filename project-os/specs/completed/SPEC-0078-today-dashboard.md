# Summary

Stock Master の起動直後に「今日やること」を判断できるダッシュボードを追加する。既存の3モードは維持しつつ、毎日の起点として、献立、期限食材、買い物、未同期、料理開始を一画面に集約する。

## 背景

現状は食材管理、献立・レシピ、料理・記録が整理されているが、日常利用では「今日は何を確認すべきか」が複数画面に分散している。レシピ/食材管理アプリでは、ホーム画面が献立・買い物・期限アラートをまとめるほど継続利用しやすい。

## 仕様

- ダッシュボードは `state.inventory`, `state.shopping`, `state.schedule`, `state.recipes`, `state.cookingHistory`, `state.pendingSync` から派生表示する。
- 表示ブロック:
  - 今日の献立: 今日の日付の未完了スケジュールを朝/昼/晩で表示し、料理開始・レシピ確認へ遷移する。
  - 期限が近い: 実質期限、なければ表示期限を基準に期限切れ/3日以内/7日以内を表示する。
  - 買い物: 未購入件数、材料まとめ件数、直近の出自を表示し、買い物リストへ遷移する。
  - 未同期: `getPendingSyncCount()` 相当の件数を表示し、同期バーと矛盾しない案内を出す。
  - 作る候補: 今日の未予定時に、期限食材・よく作る/しばらく作っていない料理から候補を出す。
- 導線は既存関数を再利用する。新規保存や新規GAS通信は追加しない。
- 配置案:
  - 案A: 起動後の初期モードを新しい「今日」画面にする。
  - 案B: ボトムナビは3項目維持し、モードC通常表示の上部に「今日」セクションを追加する。
  - 実装時は画面密度と既存導線の影響を見て選ぶ。第一候補はボトムナビ4項目化より、既存3モードを保つ案B。

## 非対象

- 新規Spreadsheetシート追加
- AIの新規呼び出し
- 献立の自動確定
- 通知/リマインダー

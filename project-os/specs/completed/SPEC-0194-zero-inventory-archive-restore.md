# Summary

数量0の食材を通常一覧から消しつつ、誤操作や後からの再登録に備えて履歴から戻せるようにする。DB変更を含むため、RLSと復元動線を安全に設計する。

## 仕様

- 0在庫は「完全削除」ではなく「通常一覧から非表示」にする。
- 実装方式は次のどちらかを採用する:
  - `inventory_items` にアーカイブ用の列を追加する
  - 0在庫履歴用の新規テーブルを追加する
- 別セッション実装時は、既存の参照関係と復元のしやすさを見て方式を決め、migrationに理由をコメントする。
- 復元時は数量を入力できるようにし、0より大きい値だけ在庫へ戻す。

## 非対象

- 調理履歴そのものの取り消し仕様変更
- 在庫の完全削除履歴
- AI解析候補の変更

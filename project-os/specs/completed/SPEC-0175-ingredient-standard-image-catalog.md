# Summary

食材画像が何も無い状態を避けるため、無料利用可能な標準画像またはアイコンのカタログを作る。すべてを写真で網羅するより、まずライセンスが明確なアイコン/イラスト系を優先する。

## 背景

食材画像も固定mapや絵文字だけでは実運用上の価値が弱い。一方で、外部画像はライセンスリスクがあるため、採用元を決めて記録し、足りないものはユーザー画像で補える設計にする。

## 仕様

- `web/public/images/ingredients/` に標準画像を置く。
- `ingredient-image.ts` 相当のresolverを作る。
- 表示優先順位は後続TKT-0176で以下にする前提。
  1. ユーザー登録画像
  2. 標準食材画像
  3. 既存絵文字
- このチケットでは 2 と 3 を実装する。
- 初期カタログ候補:
  - 肉: 鶏肉、豚肉、牛肉、ひき肉、鮭
  - 野菜: 玉ねぎ、にんじん、じゃがいも、キャベツ、トマト、きのこ、小松菜、大根、長ねぎ
  - 卵・乳製品: 卵、牛乳、チーズ
  - 主食: 米、パン、パスタ、うどん
  - 調味料: 醤油、味噌、塩、砂糖、酢、みりん、酒、油、カレールー
- 「全部を完全網羅」ではなく、resolverを増やしやすい構造にする。

## 非対象

- ユーザーが食材画像をアップロードする機能（TKT-0176）。
- DB schema変更。
- ライセンス不明な外部写真の追加。

## 実装メモ

- 画像ソース候補は実装時に最新ライセンスを確認する。
- 外部依存パッケージ追加は原則避ける。追加する場合はdependency security reviewが必要。
- 画像ではなくCSS/絵文字/既存アイコンで十分な場合は、標準画像数を絞ってよい。ただし「足りないときに増やしやすいmap」を作る。

## 残リスク

- 無料素材でもライセンス条件が変わる可能性がある。
- 写真素材で全食材を揃えると容量が増えるため、最初は軽いアイコン/イラストが安全。

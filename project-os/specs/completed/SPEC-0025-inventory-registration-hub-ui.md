# Summary

食材管理の保存場所タブから作業キュー系の導線を外し、ヘッダー右上に食材管理・買い物リスト・追加のタブ型ナビゲーションを配置する。

## 背景

現状は登録待ち・買い物リストが保存場所タブに混在し、画像スキャン、手動追加、買い物追加、在庫追加の導線が画面上下に分散している。

## 仕様

- ヘッダー右上は `食材管理`、`買い物リスト`、`＋` の3タブUIにする。
- 3タブUIは、食材管理はスレート、買い物リストはエメラルド、追加はアンバーを基調にする。
- ヘッダーのタイトル周辺と下部アクション領域は色面に含めず、リスト領域だけをタブ色に連動させる。
- 保存場所タブ、買い物リスト操作、登録ステーションは固定高さの共通サブヘッダー枠に入れ、リスト開始位置を3タブで揃える。
- 買い物リスト行はエメラルド系、登録待ち行はアンバー系の背景で揃える。
- `＋` は食材入力モーダル直開きではなく、登録待ち画面へ遷移する。
- `買い物リスト` は追加画面内ではなく、ヘッダーの独立した確認画面タブとして配置する。
- `食材管理` は通常の在庫一覧へ戻るタブとして配置する。
- 保存場所タブは実保存場所と管理ボタンだけにする。
- 登録待ち画面上部に「買い物リストへ追加」系の導線を配置する。
- 登録待ち画面内には画像スキャンと手動追加だけを置き、買い物リスト確認ボタンは置かない。
- 登録待ち画面下部は「在庫へ追加」系操作として、主ボタンと補助操作に整理する。

## 非対象

- GAS URL、API payload、Spreadsheet列、同期方式の変更。
- 買い物リストをボトムナビの独立モードにする変更。

# Summary

レシピ一覧カードを「保存済みデータの一覧」から「今日どれを作るか決めるためのカード」に強化する。写真、前回調理日、評価、期限食材との一致など、既存データから派生できる判断材料をカード上に出す。

## 背景

現在のレシピカードは材料数、調理回数、登録日、ジャンルが中心。献立選びでは、写真、前回作った日、評価、期限食材を使えるかがより意思決定に効く。料理履歴と在庫データが既にあるため、保存形式を増やさず改善できる余地が大きい。

## 仕様

- レシピごとに派生メタを計算する。
  - 前回調理日: `recipe.history` または `state.cookingHistory` から取得する。
  - 評価サマリー: 料理履歴の平均/最新評価を表示する。評価なしは非表示。
  - 写真: 料理履歴に写真がある場合、最新写真をサムネイル候補にする。
  - 期限食材一致: レシピ材料と期限が近い在庫の品名/単位が一致する場合に表示する。
  - しばらく作っていない: 前回調理日が一定期間以上前のものに表示する。
- カード表示案:
  - 左に写真サムネイルまたは既存アイコン。
  - 中央にレシピ名、ジャンル、主要メタ。
  - 右に既存アクションを維持。
  - バッジは最大2〜3個に制限する。
- レシピビューア上部にも、前回調理日、評価、作る候補/献立追加導線を整理して表示する。

## 非対象

- レシピ画像の新規アップロード機能
- 栄養/カロリー
- お気に入りの永続化。ただし作りたい候補キューとの連携は TKT-0079 で扱う

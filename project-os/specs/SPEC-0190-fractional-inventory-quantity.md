# Summary

食材数量で分数を扱えるようにする。DBは既存の numeric を使い、入力文字列だけを安全に数値へ変換する。
表示はできるだけ分数に戻し、料理で一部消費した時の「残り」が直感的に分かるようにする。

## 仕様

- 対象はWeb版 `web/`。Canvas版は編集しない。
- 入力:
  - `NumberField` と数量正規化ヘルパーで、半角/全角の数字・小数・分数を扱う。
  - `1/2` は `0.5`、`1/3` は約 `0.333333` としてDBへ保存する。
  - 不正値は保存前に止める。
- 表示:
  - 代表的な分数（1/2, 1/3, 2/3, 1/4, 3/4 など）は分数表記を優先する。
  - 分数に戻せない値は小数を短く丸めて表示する。
- 減算:
  - 調理完了時と在庫一覧の数量操作で、端数を丸めすぎない。
  - 在庫をマイナスにはしない。

## 非対象

- DBカラム追加・型変更
- 単位換算仕様の全面見直し
- 0在庫の履歴復元（TKT-0194）

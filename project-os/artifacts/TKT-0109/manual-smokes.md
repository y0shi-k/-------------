# TKT-0109 手動確認メモ

## 確認対象

- Web版ホームの「レシピ・献立・買い物」セクション。
- 対象テーブルは `recipes`、`recipe_ingredients`、`meal_schedules`、`shopping_items`、`cooking_history`。

## 手動確認手順

1. ログイン後、ホームに「レシピ・献立・買い物」が表示される。
2. レシピ名、材料、下準備、調理手順を入力して保存する。
3. 保存したレシピが一覧と詳細に表示される。
4. レシピの「編集」から内容を更新できる。
5. 日付、食事、レシピを選び「献立に追加」を押す。
6. 献立を選ぶと不足食材が表示され、チェックした食材だけ買い物へ追加される。
7. 献立の「調理完了」を押すと、献立が完了になり料理履歴に記録される。
8. スマホ幅で、入力欄、材料行、献立カード、買い物ボタンが横にはみ出さない。

## 自動確認済み

- レシピ作成、編集、献立追加、不足食材の買い物追加、調理完了から料理履歴作成をテストで確認した。
- Web版にGAS、Spreadsheet、Drive依存が追加されていないことを確認した。
- 秘密鍵の実値は追加していない。

## 残リスク

- 実Supabase環境でのRLS実動作は、ログイン済みユーザーで手動確認が必要。
- 不足判定は同名・同単位のみ比較するため、g/ml/個などの単位換算は未対応。

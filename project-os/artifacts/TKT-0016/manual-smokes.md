# TKT-0016 手動スモークテスト

## 前提
- `app.html` を Gemini Canvas に貼り付けてプレビュー
- GAS_URL は正しく設定済みであること

## テスト手順

### 1. 分類選択肢拡張確認
1. 在庫管理（モードA）→ ＋ボタンで登録フォームを開く
2. 「分類」セレクトを開く
3. **期待結果**: 選択肢に「食材」「野菜」「肉」「魚」「乳製品」「穀物」「調味料」「その他」が含まれる

### 2. 食材選択モーダル UI 確認
1. AI献立・レシピ（モードB）→ 「🎯 指定食材から」ボタンを押す
2. **期待結果**: モーダル上部に以下が表示される
   - 検索欄（placeholder: 食材名で検索...）
   - ソートボタン群（名前 / 購入日 / 期限）
   - 保存場所フィルタセレクト
   - ジャンル（分類）フィルタセレクト

### 3. 検索と選択保持（最重要）
1. 検索欄に「キャベツ」と入力し、キャベツにチェックを入れる
2. 検索欄を「にん」と変更
3. **期待結果**: キャベツのチェックが保持されており、「選択済み」エリアにキャベツのタグが表示されたまま
4. にんじんにもチェックを入れる
5. **期待結果**: 「選択済み」にキャベツ・にんじんのタグが両方表示される
6. 検索欄を空にする
7. **期待結果**: キャベツ・にんじんのチェックが両方保持されている

### 4. タグからの解除
1. 「選択済み」エリアのキャベツタグの × ボタンを押す
2. **期待結果**: キャベツのタグが消え、リストのキャベツのチェックも外れている

### 5. すべて解除
1. さらにいくつか食材にチェックを入れる
2. 「すべて解除」ボタンを押す
3. **期待結果**: すべてのチェックが外れ、「選択済み」エリアが非表示になる

### 6. ソート・フィルタとの併用
1. いくつか食材を選択する
2. ソートを「名前」に変更
3. **期待結果**: 選択状態が維持される
4. 保存場所フィルタを変更
5. **期待結果**: 選択状態が維持される（フィルタで消えた食材は見えないが、選択は保持）
6. フィルタを「すべて」に戻す
7. **期待結果**: 選択していた食材のチェックが復活している

### 7. 件数表示
1. 3つ食材を選択する
2. **期待結果**: 「レシピを考案」ボタンが「レシピを考案 (3)」と表示される
3. すべて解除
4. **期待結果**: ボタンが「レシピを考案」に戻る

### 8. 在庫管理側との干渉なし
1. モーダルを閉じる
2. 在庫管理側で「名前」ソートに変更
3. 再度「指定食材から」を開く
4. **期待結果**: モーダルのソートは「期限」のまま、選択状態はリセットされている

## 判定基準
上記すべての期待結果が満たされれば PASS。

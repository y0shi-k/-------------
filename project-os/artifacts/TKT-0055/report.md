# Report

## 変更目的

Gemini API から `401` が返された際に、技術的なエラーメッセージではなく「APIの利用制限に達した可能性があります。しばらくしてから実行してください。」というユーザー向けメッセージを表示する。

## 変更箇所

### `app.html`

#### 1. `parseRecipeTextWithAI()` catch ブロック
- `err.message.includes('401')` で判定
- 401 時: `APIの利用制限に達した可能性があります。しばらくしてから実行してください。`
- その他: 既存メッセージ `レシピの解析中にエラーが発生しました。` を維持

#### 2. `consultAiRecipe()` catch ブロック
- 同様の 401 判定・専用メッセージ追加
- その他: 既存メッセージ `相談中にエラーが発生しました。` を維持

#### 3. `generateAiRecipeFromPlan()` catch ブロック
- 同様の 401 判定・専用メッセージ追加
- その他: 既存メッセージ `レシピの生成中にエラーが発生しました。` を維持

#### 4. `scanImageWithAI()` catch ブロック
- 同様の 401 判定・専用メッセージ追加
- その他: 既存メッセージ `画像の解析中にエラーが発生しました。` を維持

#### 5. `batchPredictAI()` — 2 重の修正
- **追加**: `fetch` 直後に `!res.ok` チェック（他の Gemini API 関数と同じパターン）
  - 401/403/429/500 などの HTTP エラー時に `throw new Error('APIエラー: ' + res.status)`
- **修正**: `catch` ブロックに 401 判定・専用メッセージを追加
  - その他: 既存メッセージ `AI解析中にエラーが発生しました。` を維持

## 影響範囲

- 既存の非 401 エラーメッセージは変更なし
- スプレッドシート・GAS 通信には影響なし（AI クライアントサイドのみ）
- UI コンポーネント・Tailwind クラスに変更なし
- `GEMINI_API_KEY` の空チェックは追加していない（既存パターン維持）

## 残リスク

- 401 以外の一時的エラー（429, 503 など）には今回の専用メッセージは適用されない
  - 必要であれば別途対応

# TKT-0148 手動スモーク

## 実施結果

- `npm run dev` でローカル起動確認。
  - 3000番は使用中だったため、Next.js が `http://localhost:3002` を使用。
- in-app Browser で `http://localhost:3002` を開き、ログイン画面が正常表示されることを確認。
- ログイン後の献立データを使った実操作は、この環境では未実施。

## 自動テストで確認した内容

- 献立カードから「調理を開始」し、全画面調理ビューア下部の「料理を完了する」で消費量モーダルが開く。
- 消費量モーダルにカテゴリタブ、全選択、全解除、在庫不足警告が出る。
- チェックを外した行は在庫減算と消費イベント保存の対象外になる。
- 評価とコメントが `cooking_history` へ保存される。
- 写真なしでも完了できる。

## 未実施の手動確認

- 実端末カメラでの `capture="environment"` 起動。
- 実 Supabase Storage への完成写真アップロード。
- 料理・記録画面で、保存済み完成写真が署名URL経由で表示されること。

## 補足

未実施項目は、ログイン済みユーザー、献立データ、Storage接続、端末カメラが必要なため、実環境での最終確認項目として残す。

# TKT-0024 Manual Smoke Tests

## 事前準備

1. `app.html` を Gemini Canvas に貼り付けてプレビュー
2. ブラウザの開発者ツールを開いておく（Consoleタブ）

## テストケース

### Test 1: フォルダ作成とスプシ移動

**手順:**
1. 「アプリを起動」をクリック
2. GDriveを開き、ルート直下に「料理レシピ・食材管理_Canvasアプリ」フォルダが作成されたことを確認
3. そのフォルダ内に「料理レシピ・食材管理DB」スプシが存在することを確認

**期待結果:**
- フォルダが作成される
- スプシがフォルダ内に配置される
- アプリが正常に起動し、在庫管理画面が表示される

### Test 2: 最終編集日時カラムの確認

**手順:**
1. スプシを開く
2. 各シート（食材在庫、レシピ集、献立スケジュール、料理履歴、買い物リスト）の1行目を確認

**期待結果:**
- 全5シートの最後の列に「最終編集日時」ヘッダーが存在する

### Test 3: 在庫追加時の最終編集日時

**手順:**
1. 「＋」ボタンから食材を追加（例: 牛乳 1L）
2. 「同期する」ボタンをクリック
3. スプシの「食材在庫」シートを確認

**期待結果:**
- 追加された行の最後の列に `yyyy-MM-dd HH:mm:ss` 形式の日時が入っている

### Test 4: レシピ追加時の最終編集日時

**手順:**
1. モードB（献立・レシピ）に切り替え
2. 「新規レシピ」からレシピを追加
3. 「同期する」ボタンをクリック
4. スプシの「レシピ集」シートを確認

**期待結果:**
- 追加された行の最後の列に日時が入っている

### Test 5: 献立登録時の最終編集日時

**手順:**
1. 献立スケジュールにレシピを登録
2. 「同期する」ボタンをクリック
3. スプシの「献立スケジュール」シートを確認

**期待結果:**
- 追加された行の最後の列に日時が入っている

### Test 6: 買い物リスト→購入時の最終編集日時

**手順:**
1. 買い物リストに手動追加
2. 「同期する」で追加
3. チェックして「購入済にする」
4. 「同期する」をクリック
5. スプシの「買い物リスト」と「食材在庫」を確認

**期待結果:**
- 買い物リストの該当行の最後の列に日時が入っている
- 食材在庫に移行された行の最後の列にも日時が入っている

### Test 7: 料理完了時の写真保存先

**手順:**
1. 献立スケジュールからレシピを選択して「料理を開始」
2. 料理完了時に写真を撮影/選択
3. 「料理を完了する」
4. 「同期する」をクリック
5. GDriveの「料理レシピ・食材管理_Canvasアプリ」フォルダ内に写真ファイルが保存されたことを確認
6. スプシの「料理履歴」シートの最後の列に日時が入っていることを確認

**期待結果:**
- 写真がフォルダ内に保存される
- 料理履歴の最後の列に日時が入っている

### Test 8: 在庫更新時の最終編集日時

**手順:**
1. 既存の在庫アイテムを編集（数量変更など）
2. 「同期する」をクリック
3. スプシの「食材在庫」シートを確認

**期待結果:**
- 更新された行の最後の列に新しい日時が入っている

### Test 9: 既存読み出しの動作確認

**手順:**
1. ページをリロード
2. 「アプリを起動」をクリック
3. 各モード（A/B/C）を切り替えてデータが正しく表示されることを確認

**期待結果:**
- 在庫、レシピ、献立、買い物リストが正常に読み込まれる
- データの欠損やずれがない

## 補足

- 既存スプシが存在する場合、新規スプシが別IDで作成される（仕様通り）
- 既存フォルダが存在する場合はそのフォルダが使用される

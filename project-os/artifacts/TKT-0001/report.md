# 変更報告書（Stock Master）

## 変更目的
在庫一覧がカード型で1画面に入らず、期限情報も見えないという課題を解消するため、コンパクトな行リスト型に変更しました。同時にソート・編集・数量調整・期限ハイライト機能を追加し、日々の在庫管理の効率を向上させます。

## 今回追加した安全装置
- **二重送信防止**: `setStatus()` によるグローバル disabled 制御を新規ボタン群にも継承
- **削除確認ダイアログ**: `confirm()` で誤削除防止
- **数量下限制御**: `Math.max(0, ...)` で在庫数量が負数にならない
- **スキーマ厳守**: Google Spreadsheet のカラム定義を一切変更せず、`setValue` / `getValues` の順序を既存スキーマに厳密に合わせた

## 実施した確認
- HTML構文チェックにパス
- `executeGAS` / `GAS_URL` / `setStatus` が残存・機能することを確認
- 新規Tailwindクラスが既存UIパターンと一致することを確認
- スキーマ整合性を確認（カラム追加・削除・リネームなし）

## 残リスク
- GAS側の関数実行はローカル環境で検証できていない（デプロイ後の手動確認が必要）
- `g` / `ml` 単位の場合、±ボタンは50ずつ増減する（1ずつでは使いにくいため）。その他の単位は1ずつ。
- 期限のハイライトはブラウザの `new Date()` で判定しているため、端末の日時設定に依存する

## 次の依頼や人判断
- GAS Webアプリへ再デプロイして実際のスプレッドシート連携を確認してください
- スマホ実機で操作感（タップしやすさ、誤タップ）を確認し、フィードバックがあれば調整します
- 期限ハイライトの色合い（赤・黄・通常）が見やすいか、実データで確認してください

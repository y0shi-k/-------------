# TKT-0022 Manual Smoke Tests

## 1. フロー全体
- [ ] 献立スケジュールからレシピを選び、料理ビューアを開く
- [ ] 「料理を完了する」→ 消費量調整 → 確定 → 「料理の記録」モーダルが表示される
- [ ] 「記録を保存」で一括更新が実行される（処理中オーバーレイが表示される）
- [ ] 成功後「料理が完了しました！」のToastが表示される

## 2. 在庫更新
- [ ] 消費量調整で入力した分だけ在庫が減算される
- [ ] 在庫が0になった食材は削除される
- [ ] スプレッドシート「食材在庫」シートに反映される

## 3. 献立ステータス更新
- [ ] 今日の献立スケジュールで該当レシピのステータスが「完了」に更新される
- [ ] スプレッドシート「献立スケジュール」シートに反映される

## 4. レシピ集更新
- [ ] 該当レシピの調理回数が+1される
- [ ] 調理日履歴に今日の日付が追加される
- [ ] スプレッドシート「レシピ集」シートに反映される

## 5. 料理履歴追加
- [ ] 「料理履歴」シートに新規行が追加される
- [ ] レシピID、レシピ名、調理日時、評価、感想、写真URL、履歴IDが正しく記録される

## 6. エラーハンドリング
- [ ] 在庫不足時もcompleteRecipeが正常に完了する
- [ ] GAS通信失敗時「保存に失敗しました。再度お試しください。」が表示される
- [ ] 献立スケジュールに該当予定がない場合（直接レシピ集から料理した場合）も正常に完了する

## 7. 制約チェック
- [ ] alert / confirm / prompt が使用されていない
- [ ] スプシ書き込み系は executeGAS 経由（syncPendingChanges 以外でも一括更新として許容）
- [ ] executeGAS にオブジェクトを渡しても正常に動作する（JSON.stringify 自動化）

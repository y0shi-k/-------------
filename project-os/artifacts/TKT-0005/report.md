# TKT-0005 完了報告 — AIレシピ考案機能

## 変更目的

モードB（献立・レシピ）に「AIレシピ考案」機能を追加しました。在庫の期限が迫っている食材を優先的に使うレシピ、またはユーザーが選んだ食材から作れるレシピを、Gemini AIが提案します。生成したレシピは「レシピ集」に保存でき、献立スケジュールへの割り当て（今後実装）につなげられます。

## 今回追加した安全装置

- **フリーズ防止**: AI通信中は `setStatus()` で画面をロックし、二重送信を防ぎます
- **エラートースト**: APIエラー時に `showToast()` で通知し、操作可能な状態に戻ります
- **スプシ安全**: レシピ保存は `pendingSync` に積んで手動一括同期。操作ごとの即時GAS通信はしません
- **数量問題対策**: AIに「在庫を全部使い切れ」と伝わらないよう、数量表記をプロンプトから削除
- **プロンプト編集**: AI提案プロンプトをユーザーが確認・修正できるため、意図しないレシピを事前に防げます
- **2段階確認**: 「相談（方針確認）→ 生成」で、レシピ生成前に方向性を合意形成できます

## 実施した確認

- HTML構文チェック: パス
- GAS通信パターン: 既存パターン維持
- UI整合性: 既存のモーダル・ボタン・バッジデザインを踏襲
- Canvas制約: alert/confirm/prompt なし、APIキー空チェックなし
- 実際のGAS連携: 保存フローは `queueRecipeCreate()` → `syncPendingChanges()` の既存パスを流用

## 残リスク

- **Gemini APIキー未設定**: Canvasに貼り付け後、ユーザーが `GEMINI_API_KEY` にキーを入力する必要があります（空のままではAI通信がエラーになります）
- **AIの解釈ミス**: 「1パック=10個」などの換算はAIに委ねており、人間の感覚とズレることがあります。プロンプト編集機能で補正可能です
- **2回API通信**: 相談→生成で2回APIを叩くため、Gemini APIのレート制限や通信環境によっては遅延が生じることがあります

## 次の依頼や人判断

- Canvasに貼り付けて、実際にGemini APIキーを設定して動作確認をお願いします
- AIレシピ考案の使い心地（プロンプト編集の敷居、2段階フローの快適さ）をフィードバックください
- 問題なければ **TKT-0006（献立スケジュール）** へ進めます

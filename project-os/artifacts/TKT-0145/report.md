# TKT-0145 実装レポート

## 結論

テキストから追加／AI考案のAI処理後に、中間プレビューを挟まずレシピ編集モーダルを直接開くように変更した。テキストから追加モーダルは開くたび、また閉じるたびに本文をクリアする。

## 変更内容

- `aiPreview` state と中間プレビュー表示を削除した。
- `runAiRecipe` はAI結果を返すだけにし、成功時は `applyRecipeToEditor` で編集モーダルへ流し込む構造にした。
- テキスト構造化、優先消費レシピ、指定食材から考案、既存の互換AIパネルの各導線で、成功後に直接編集モーダルを開くようにした。
- `テキストから追加` の本文欄が前回内容を残さないよう、開く時と閉じる時に `aiSourceText` をリセットした。
- 未使用になった `.ai-preview` CSS を削除した。
- レシピAI導線のテストを更新し、直接編集モーダルが開くこと、確認ボタンが出ないこと、本文欄が再表示時に空になることを追加確認した。

## セキュリティ

- `/api/ai/recipes` 経由のまま変更しており、Gemini APIキーをブラウザへ出す変更はない。
- schema、auth、RLS、Storage、データ削除処理は変更していない。

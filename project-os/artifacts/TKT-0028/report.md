# 変更目的

Driveの写真リンクは別タブでは開ける一方、`<img>` では表示できないため、GASで写真をBase64化して返し、料理履歴カード内で表示できるようにしました。

# 今回追加した安全装置

- Driveファイルの共有設定は変更していません。
- スプレッドシートやDriveへの書き込み処理は追加していません。
- 取得済み画像は `state.cookingHistoryImageCache` に保持し、同じ画面での重複取得を避けます。
- 初期同期後と一括同期後に、直近6件の写真を裏で先読みします。
- 起動を早くするため、先読みは同期完了から4秒後、可能ならブラウザのアイドルタイミングで開始します。
- 写真取得はステータスオーバーレイを出さず、画面操作を止めません。
- 先読み完了時に料理履歴タブを開いていれば、自動で再描画して写真表示へ切り替えます。
- Base64取得に失敗した場合も「写真を開く」リンクは残ります。

# 実施した確認

- 標準 verify: `VERIFY_PASSED`
- JavaScript構文チェック: `JS_PARSE_PASSED`
- `setSharing` / `ANYONE_WITH_LINK` が追加されていないことを確認
- `alert` / `confirm` / `prompt` がないことを確認
- TKT-0028のGAS通信が読み取り専用であることを確認
- 先読み処理が `executeGAS(..., { silent: true })` で動くことを確認
- 先読みが `scheduleCookingHistoryImagePrefetch()` で遅延・アイドル実行されることを確認

# 残リスク

- 実際の画像表示はGASデプロイ環境とDrive権限に依存するため、Canvasプレビューで確認が必要です。
- 写真が多い場合は初期先読み対象外の写真が最大6件ずつ順次読み込まれるため、全件表示まで少し時間がかかる可能性があります。

# TKT-0078 Manual Smokes

ブラウザ実表示はユーザー実施。実装者側では以下の観点で静的に導線と空状態を確認した。

- 今日の未完了献立あり: `getTodayScheduleItems()` が `state.schedule` から今日かつ未完了のみを朝/昼/晩順で抽出し、`openCookingViewer()` と `openRecipeViewer()` へ接続する。
- 今日の未完了献立なし: 献立カードは空状態を表示し、作る候補カードを表示する。
- 期限切れ/3日以内/7日以内: `getExpiryInventoryRows()` の既存分類を再利用し、期限食材カードへ最大3件を表示する。
- 期限食材なし: 「期限が近い食材はありません」の空状態を表示する。
- 買い物あり/なし: `state.shopping` から件数、材料まとめ件数、直近出自を派生表示し、0件でもカードを維持する。
- pendingSync あり/なし: `getPendingSyncCount()` を参照し、同期バーと同じ件数を表示する。
- 候補レシピあり/なし: レシピ/履歴/期限食材からローカル候補を最大3件表示し、候補なしでも空状態を表示する。
- 遷移: 献立からクッキングビューア、期限食材から使い切り/レシピ検索、買い物カードから買い物リストへ既存関数で遷移する。

# TKT-0023 Manual Smoke Tests

## 1. 料理完了フロー全体
- [ ] 献立スケジュールからレシピを選び、料理ビューアを開く
- [ ] 「料理を完了する」→ 消費量調整 → 確定 → 「料理の記録」モーダルが表示される
- [ ] 「記録を保存」でモーダルが閉じる
- [ ] 未同期バーに「在庫 N件 / 料理履歴 1件」などのカウントが表示される

## 2. 手動一括同期
- [ ] 「同期する」ボタンを押すと、未同期バーのカウントが0になる
- [ ] 処理中オーバーレイが表示される
- [ ] 成功後「未同期の変更をスプレッドシートへ反映しました。」のToastが表示される

## 3. スプシ反映確認
- [ ] 食材在庫シート: 消費量調整で入力した分だけ在庫が減算されている
- [ ] 献立スケジュールシート: 該当予定のステータスが「完了」になっている
- [ ] レシピ集シート: 該当レシピの調理回数が+1、履歴に今日の日付が追加されている
- [ ] 料理履歴シート: 新規行が追加され、レシピID・レシピ名・調理日時・評価・感想が正しい

## 4. 画像アップロード（一括同期時）
- [ ] 写真を選択して「記録を保存」→ 一括同期
- [ ] Driveに画像ファイルが保存されている
- [ ] 料理履歴シートの写真URL列にDriveリンクが記録されている
- [ ] 画像未選択時も正常に完了する（URLは空文字）

## 5. エラーハンドリング
- [ ] 在庫不足時も正常に完了する（在庫分だけ減算）
- [ ] 画像アップロード失敗時も記録が継続される（URLは空文字）
- [ ] 献立スケジュールに該当予定がない場合も正常に完了する

## 6. 制約チェック
- [ ] alert / confirm / prompt が使用されていない
- [ ] すべてのスプシ書き込みは `syncPendingChanges()` 経由（画像アップロードも含む）

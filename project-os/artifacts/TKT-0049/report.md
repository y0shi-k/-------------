# TKT-0049 Report

status: ready

## 今回の変更目的

料理ビューアの戻るボタンで、常に料理履歴へ戻るのではなく、開いた元の画面へ戻れるようにした。

## これまで不足していた点

`openCookingViewer()` が起動元を保持していなかったため、スケジュール画面やレシピ集から開いても、戻る時にモードC側の通常表示へ戻っていた。

## 今回追加した安全装置

料理ビューア起動時に戻り先を `state.cookingReturnContext` へ保存する。スケジュールからならスケジュールタブ、レシピ集からならレシピ集タブ、料理履歴からなら料理履歴へ戻す。

## 実施した確認

- HTML構文チェックと `executeGAS` / `GAS_URL` 存在確認: PASS
- スケジュール/レシピ集/料理履歴の各起動ボタンが戻り先を渡すことを確認
- 新規のSpreadsheet書き込みやGAS通信追加がないことを確認

## まだ人判断が必要な残リスク

Canvasプレビュー上で、実際に各入口からビューアを開いて戻るボタンの遷移を確認する必要がある。

# Manual Smokes — TKT-0058-consumption-substitution-tabs

## 前提条件
- アプリを起動し、在庫に「豆腐（2丁）」「絹豆腐（1丁）」などの同系統食材が登録されていること
- レシピに材料（豆腐 1丁）と調味料（醤油 大さじ1）が登録されていること

## 手順

### 1. タブ切り替え
1. モードC（料理・記録）でレシピを選択し、料理ビューアを開く
2. 「料理を完了する」ボタンを押す
3. **期待結果**: 調整モーダルが開き、上部に「All」「調味料」「材料」の3タブが表示される
4. **期待結果**: 初期状態で「All」タブが選択され（白背景・indigo文字）、材料と調味料が混在して表示される
5. 「調味料」タブを押す
6. **期待結果**: 調味料のみ表示され、材料行は非表示になる
7. 「材料」タブを押す
8. **期待結果**: 材料のみ表示される
9. 数量を入力し、タブを切り替える
10. **期待結果**: 入力した数量が保持される

### 2. 代替品選択
1. 材料行（例: 豆腐）の「代替」ボタンを押す
2. **期待結果**: 「代替品を選択」モーダルが開き、在庫食材一覧が表示される
3. 検索バーに「絹」と入力する
4. **期待結果**: 絹豆腐のみフィルタされて表示される
5. 保存場所フィルタや分類フィルタを変更する
6. **期待結果**: リストが即時フィルタされる
7. 期限ソートを「名前」に切り替える
8. **期待結果**: 名前順に並び替えられる
9. 絹豆腐の行をタップする
10. **期待結果**: モーダルが閉じ、元の調整画面で「豆腐 → 絹豆腐」と表示される
11. **期待結果**: 在庫表示が「絹豆腐」の在庫数量に更新される

### 3. 元に戻す
1. 代替設定された行の「元に戻す」リンクを押す
2. **期待結果**: 表示が「豆腐」に戻り、在庫数も豆腐の在庫数に戻る

### 4. 確定処理
1. 代替設定した状態で数量を入力し、「確定」を押す
2. **期待結果**: 絹豆腐の在庫が減算される（メッセージに「豆腐→絹豆腐」と表示される）
3. **期待結果**: 同期バーに未同期件数が表示される
4. **期待結果**: 料理の記録モーダルが自動で開く

### 5. エッジケース
1. 代替品の在庫が0の状態で確定する
2. **期待結果**: 「在庫不足（0丁しかない）のため、全在庫を消費しました」と表示される
3. 数量を0にして確定する
4. **期待結果**: 該当材料は在庫減算されない

## 非回帰確認
- 従来通り、代替を設定しないまま確定しても正しく在庫が減算される
- タブ切り替えなし（Allのまま）で確定しても動作する
- 買い物リスト・食材管理・献立スケジュールの各画面に影響がない

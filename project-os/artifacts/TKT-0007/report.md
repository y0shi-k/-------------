# 変更報告書（Stock Master）

## 変更目的
献立にレシピを割り当てた際、またはレシピ集から直接、必要な材料と現在の食材在庫を自動比較して、足りないものを買い物リストに追記できるようにしました。これにより、「献立を決めたら買い物リストが自動で完成する」体験を提供します。

## 今回追加した安全装置
- **手動一括同期の継承**: 買い物リストへの追記は `state.pendingSync.shoppingCreates` に積み、`同期する` ボタン1回でスプレッドシートに反映。同名・同単位の未購入アイテムがあれば数量を加算し、重複を防ぐ
- **在庫比較の安全性**: 在庫の品名＋単位がレシピ材料と完全一致した場合のみ比較。単位の異なるものは別アイテムとして扱い、誤った差分計算を防ぐ
- **ワンタップ遷移**: 不足材料が買い物リストに追加された後、「買い物リストを確認」ボタン1つでモードAの買い物タブへ瞬時に移動できる
- **スキーマ厳守**: `買い物リスト` シートのカラム定義を一切変更せず、`appendRow` の順序を既存スキーマに厳密に合わせた

## 実施した確認
- HTML構文チェックにパス
- `executeGAS` / `GAS_URL` / `setStatus` が残存・機能することを確認
- 買い物追記系の操作が `syncPendingChanges()` 以外で個別GAS通信されていないことを確認
- `alert/confirm/prompt` が残存していないことを確認
- `showToastWithAction` が新規追加され、DOMベースの通知を維持していることを確認

## 残リスク
- GAS側の買い物リスト追記・数量加算はローカル環境で検証できていない（デプロイ後の手動確認が必要）
- 在庫とレシピ材料の名前・単位が完全一致でない場合、不足判定されない（「人参」と「にんじん」、gと個の換算等は未対応）
- レシピ材料の `amount` が数値として解釈できない文字列の場合、`NaN` になり常に不足と判定される可能性がある

- 同期ボタンを押して、買い物リストシートに正しく追記・加算されるか確認してください

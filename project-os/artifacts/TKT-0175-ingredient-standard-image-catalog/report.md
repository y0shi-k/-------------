# TKT-0175 食材標準画像カタログ 実装レポート

## 変更内容

- `web/public/images/ingredients/` に自作SVGの標準アイコンを追加した。
- `web/src/lib/ui/ingredient-image.ts` を追加し、食材名の表記ゆれを吸収する標準画像resolverを実装した。
- `IngredientIcon` を標準画像優先、未対応食材は既存絵文字へフォールバックする表示に変更した。
- 在庫一覧では在庫アイテムのカテゴリも渡し、調味料の汎用名でも標準アイコンに寄せられるようにした。
- 標準アセットの出典と利用条件を `docs/ingredient-standard-images.md` に記録した。
- 追加確認で見つかった `牛乳` の誤分類を修正し、納豆・お茶の標準アイコンを追加した。

## ライセンス確認

- 外部画像、スクレイピング画像、商用不可素材は使用していない。
- 追加アセットはCodexによる自作SVGのため、第三者素材由来の追加ライセンス条件はない。

## 検証

- `harness/bin/verify_web.sh TKT-0175-ingredient-standard-image-catalog`: pass
- スマホ幅390pxで在庫一覧を確認し、標準画像・食材名・数量操作の座標重なりがないことを確認した。
- 同じ画面で、牛乳が牛肉ではなく乳製品アイコンに解決されることを確認した。
- 同じ画面で、納豆・伊右衛門がお茶/納豆の標準画像に解決されることを確認した。

## 残リスク

- 写真ではなく標準アイコンのため、細かい品種の見分けはTKT-0176以降のユーザー画像で補う。
- 食材名の表記ゆれは完全網羅ではないため、運用で見つかった名前をresolverに追加する。
- スマホ幅の在庫カードは重なりはないが、情報量が多いカードではまだ窮屈に見える。カード全体の再設計は別TKTで扱う。

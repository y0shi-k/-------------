# TKT-0017 Manual Smoke Tests

## Environment
- Target: `app.html` (Gemini Canvas preview)
- Mode: Mode B (献立・レシピ) > レシピ集タブ

## Test Cases

### TC-001: レシピカードタップ → 閲覧画面
**Precondition:** レシピが1件以上登録されている
**Steps:**
1. モードB「献立・レシピ」に切り替える
2. 「レシピ集」タブを開く
3. 任意のレシピカード全体をタップする（編集アイコン以外）
**Expected:**
- `aiRecipePreviewModal` が開く
- モーダルタイトルが「レシピ詳細」になっている
- レシピ名・材料・手順が読み取り専用で表示されている
- 下部ボタンが「閉じる」「編集する」になっている

### TC-002: 閲覧画面 → 編集画面
**Precondition:** TC-001 完了後
**Steps:**
1. 「編集する」ボタンをタップする
**Expected:**
- 閲覧画面が閉じる
- `recipeModal`（編集画面）が開く
- レシピ名・材料・手順が編集可能な入力欄として表示される

### TC-003: 編集アイコン → 直接編集画面
**Precondition:** レシピが1件以上登録されている
**Steps:**
1. レシピカード右端の編集アイコン（✎）をタップする
**Expected:**
- 閲覧画面を経由せず、直接 `recipeModal` が開く

### TC-004: AI生成レシピプレビュー → 従来のボタン維持
**Precondition:** 在庫に食材が登録されている
**Steps:**
1. モードB「献立・レシピ」に切り替える
2. 「AI考案」タブを開く
3. 「優先消費レシピ」または「指定食材から」でレシピを生成する
4. 生成完了後、プレビュー画面を確認する
**Expected:**
- モーダルタイトルが「生成されたレシピ」になっている
- 下部ボタンが「キャンセル」「再生成」「レシピ集に保存」になっている

### TC-005: AI生成レシピ保存後 → 閲覧画面で確認
**Precondition:** TC-004 完了後
**Steps:**
1. 「レシピ集に保存」ボタンをタップする
2. 「レシピ集」タブに切り替える
3. 先ほど保存したレシピカードをタップする
**Expected:**
- 閲覧画面（レシピ詳細）が開く
- 保存したレシピ名・材料・手順が正しく表示される
- 下部ボタンが「閉じる」「編集する」になっている

# TKT-0033 手動スモークテスト

## 前提

- `app.html` をGemini Canvasに貼り付けてプレビュー
- GASがデプロイ済みで、`SS_ID_RECIPE_APP` が設定済み
- 既存の献立スケジュールデータがある場合、**次回 `handleInit` 実行時に自動移行される**

## テスト手順

### 1. 初期化＋既存データ移行

1. アプリを起動し、`handleInit` を実行
2. Google Spreadsheet の `献立スケジュール` シートを確認
3. **期待結果**: ヘッダーが `ID, 予定日, 食事区分, レシピID, レシピ名, ステータス, 表示順, 最終編集日時` になっている
4. 既存データがあれば、ID列に `sched_...` が付与され、表示順=0 になっている

### 2. 複数レシピ配置

1. モードB → スケジュールタブを開く
2. 任意の日付・食事ブロックの「＋」ボタンをタップ
3. レシピを選択して追加
4. 同じブロックの「＋」ボタンを再度タップし、別レシピを追加
5. **期待結果**: 同じブロック内に2つのレシピカードが縦に積み重なる
6. 3つ目、4つ目も追加してみる

### 3. 同ブロック内D&D並び替え

1. 複数レシピが配置されているブロックで、ドラッグハンドル（≡）を掴む
2. 同じブロック内で上下にドラッグして別の位置にドロップ
3. **期待結果**: カードの順序が変わる
4. 上部の「同期する」ボタンを押して同期
5. ページをリロードし、献立スケジュールを再読み込み
6. **期待結果**: 並び替えた順序が維持されている

### 4. 別ブロックD&D移動

1. 朝のブロックにあるレシピカードをドラッグ
2. 昼または別日のブロックにドロップ
3. **期待結果**: レシピが移動先ブロックに表示される。元のブロックから消える
4. 「同期する」ボタンを押す
5. リロード後、移動が維持されていることを確認

### 5. 選択モード＋一括削除（未完了のみ）

1. スケジュールタブ上部の「選択モード」ボタンをタップ
2. 複数のレシピカードにチェックを入れる
3. 下部に「チェックした献立を一括削除 (N件)」ボタンが表示されることを確認
4. ボタンをタップ
5. **期待結果**: 確認モーダルなしで削除され、Toast表示

### 6. 選択モード＋一括削除（完了済み含む）

1. 完了済みの献立を1つ以上含めてチェック
2. 「チェックした献立を一括削除」をタップ
3. **期待結果**: 「完了済みの献立が含まれています。削除してもよろしいですか？」の確認モーダルが表示される
4. 「キャンセル」をタップ → 削除されない
5. 再度タップ → 「削除する」をタップ → 削除される

### 7. レシピ変更

1. 既存の献立カードをタップ（選択モードOFF時）
2. 「別のレシピに変更」を選択
3. 別のレシピを選択
4. **期待結果**: 同じ位置でレシピ名が変更される

### 8. 買い物リスト連携

1. 在庫にない材料を使うレシピを献立に追加
2. **期待結果**: 「N件の不足材料を買い物リストに追加しました」のToastが表示される

### 9. 一括同期

1. 複数の操作（追加、削除、D&D）を行う
2. 上部の「同期する」ボタンを押す
3. **期待結果**: すべての変更がスプレッドシートに反映される
4. Spreadsheet側の `献立スケジュール` シートを確認し、ID列・表示順列が正しく更新されていることを確認

## 既知の制限

- **D&DはPC（マウス操作）を前提**としている。タッチデバイスでは動作しない可能性がある
- 同ブロック内D&Dの挿入位置はマウスY座標から推定しており、カードの実際の高さと若干のずれが生じる可能性がある

# TKT-0021 Manual Smoke Tests

## 1. モーダル表示
- [ ] 料理完了 → 「料理を完了する」 → 消費量調整 → 確定後に「料理の記録」モーダルが自動で表示される
- [ ] モーダルタイトルが「料理の記録」である

## 2. 写真アップロード
- [ ] ドロップゾーンをタップして画像を選択できる
- [ ] 選択後にプレビュー画像が表示される
- [ ] 「変更」ボタンでプレビューを消して再選択できる
- [ ] 5MB以上の画像を選択するとCanvasリサイズが行われる（処理中表示が出る）

## 3. 星評価
- [ ] 初期状態は星が未選択（グレー）である
- [ ] 1〜5の星をタップで選択できる
- [ ] 選択した星までが黄色でハイライトされる

## 4. 感想入力
- [ ] 複数行テキスト入力ができる
- [ ] placeholder が正しく表示される

## 5. 保存フロー
- [ ] 画像・評価・感想を入力して「記録を保存」でモーダルが閉じる
- [ ] 画像未選択・評価未入力でも「記録を保存」でモーダルが閉じる
- [ ] 「スキップ」ボタンで何も保存せず閉じる
- [ ] 保存後、Toast「料理の記録を保存しました。」が表示される（completeRecipe未実装時）

## 6. エラーハンドリング
- [ ] GAS側で画像保存が失敗した場合、Toast「写真の保存に失敗しました。記録を続行します。」が表示され、モーダルは閉じて次へ進む

## 7. 制約チェック
- [ ] alert / confirm / prompt が使用されていない
- [ ] スプシ書き込み系は syncPendingChanges() を経由（画像アップロードを除く）

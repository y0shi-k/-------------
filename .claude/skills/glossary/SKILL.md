# /glossary — ドメイン用語・判断基準の参照

`project-os/knowledge/glossary.md` を読み、用語の意味・対応データ・判断基準に基づいて答える。

## 手順
1. `project-os/knowledge/glossary.md` を読む。
2. 質問された用語の意味・対応する Supabase テーブル・判断基準を答える。
3. 用語集に無い語なら、それを明示し、必要なら glossary.md への追記を提案する（勝手に意味を作らない）。

## 使いどころ
- 実装前に「この機能で言う『在庫』『登録待ち』は何を指すか」を固定したいとき。
- プロダクトの美学（写真認識優先・買い足しが少ない案を優先・AI結果は確定せず候補に置く）に沿っているか判断したいとき。
- 新しいドメイン用語が出てきたら glossary.md に追記して正本を育てる。
